# Customer Account 会员中心实现说明

## 后端地址解析（无需手改隧道 URL）

`src/api.ts` 通过 `resolveBackendUrl()` 在运行时推断 app 后端 base URL，不再硬编码本地隧道地址：

- 读取 `shopify.extension.scriptUrl`（扩展脚本来源地址）。
- 本地 `shopify app dev` 时，扩展脚本与 app 后端走同一个临时隧道域名（`*.trycloudflare.com` / `*.ngrok.*` / `localhost`），直接复用该 origin，每次启动地址变化也无需手动修改。
- 生产环境扩展脚本由 Shopify CDN 提供，回落到固定生产域名 `https://shopify.waterdropfilter.com`。
- 新增/更换本地隧道服务商时，在 `DEV_TUNNEL_HOST_SUFFIXES` 中补充对应域名后缀即可。

## Shopify 后台编辑器预览数据

生产环境 Shopify 后台编辑器中预览 Customer Account 扩展时，客户账户 session token / 后端鉴权可能不可用。`src/api.ts` 会在请求前判断当前运行环境是否来自后台编辑器：优先读取 Shopify 扩展运行时的 `shopify.extension.editor`，其次匹配 `admin.shopify.com` 相关的当前 URL、referrer 或 ancestor origins。若编辑器提供了不可用于真实客户的 session token，后端返回 `Invalid session token` / `Invalid customer session` 等 401/403 鉴权错误时也会回落到本地 mock 数据。

命中后台编辑器环境时，`fetchLoyaltyInit()` 和 `fetchLoyaltyTransactions()` 会直接返回 `src/editorMockData.ts` 中的本地 mock 数据，不调用 `shopify.sessionToken.get()`，也不请求 `/api/shopify-extensions/loyalty-page`。正式客户账户页面仍按原逻辑请求后端真实数据。

## Earn Points

`EarnPointsSection` 的数据来自 Smile API `/earning_rules`，由 `app/pages/api/shopify-extensions/loyalty-page.tsx` 在 init 阶段聚合返回，前端不再维护静态赚积分卡片。

- fixed 奖励展示为 `earn_pointsLabel`，variable 奖励展示为 `earn_variablePointsLabel`。
- `Celebrate a birthday`、`Place an order` 等分层规则会按当前客户 VIP tier 过滤。
- Smile 规则名通过前缀匹配到 `locales/en.default.json`、`locales/fr.json`、`locales/de.json` 中的多语言 key；新增 Smile 规则时，需要同步补充前缀映射和 locale 文案。
- 卡片排版对齐 Figma 桌面/手机稿：容器宽度大于 948px 时为双列卡片，移动端为单列；图标使用 40px 槽位，标题与积分在移动端同一行左右对齐，CTA 另起一行。

## Redeem Rewards

`RedeemSection` 的加载态与真实兑换商品共用同一响应式卡片网格，骨架卡按图片、标题/积分、进度、状态文案和 CTA 区域占位，避免数据加载完成前后布局跳动。

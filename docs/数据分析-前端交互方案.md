## 1. 设计原则(运营友好,table-first)

1. **表格即产品**:核心交付是「能看、能排序、能导出」的数据表,不是图表。趋势用多周期行 + ↑↓% 环比列表达。
2. **口径与后台一致**:指标列名沿用 Shopify Analytics / GA4 英文术语(Total Sales / Net Sales / Sessions / Session key event rate),方便运营对账;中文 tooltip 解释口径。
3. **导出无处不在**:每张表右上角「导出 Excel」;报告详情可整份导出(多 sheet)。
4. **对比直观**:环比/同比以独立列展示(↑绿/↓红 + 百分比),不藏在悬浮层。
5. **映射是数据质量的命门**:未匹配项全局红点提醒(影响数据准确),一键跳转补录。
6. **操作即反馈**:加载骨架屏;采集/生成中轮询 + 进度文案;导入校验错误行号高亮。
7. **风格一致**:沿用竞品 / Alt-Text 模块的 Polaris 排版与左侧子导航。
8. **Polaris Web Components 优先(CLAUDE.md 黄金规则 1,禁止自绘基础控件)**:页面骨架 `<s-page>` + slot 按钮 → `<s-box>` → `<s-stack>` → `<s-section>`;表格 `<s-table>` family、表单 `<s-text-field>/<s-select>/<s-checkbox>`、弹窗 `<s-modal ref>`、提示 `<s-banner>/<s-badge>/<s-text>`。Tailwind 仅限布局细节微调与 Polaris 无对应组件的场景(tabs、文件拖拽区,需注释说明)。

## 3. 页面逐一规格

### 3.1 报表 `/app/analytics`(默认页,核心)

**顶部工具栏(一行)**:

- 粒度切换:`周 / 月`(segmented)。
- 周期选择:下拉,选项来自 `GET /analytics/periods?granularity=`(如 `2026-W23(06-01 ~ 06-07)`);默认最新一期。
- 「补采本期数据」按钮(次要,触发 `POST /collect`,跑批中显示 spinner + 轮询)。
- 数据截止说明:`数据截止 2026-06-08(GA4 T+1)` 灰字。

**未匹配提醒条**(条件显示):`<s-banner tone="warning">`:"本期有 N 个商品缺 Model、M 个 GA4 商品未对齐、K 个渠道未分组,相关行计入「未映射」→ 去口径管理处理"。

**六个 Tab(`<s-tabs>`)= 六张报表**:

| Tab              | 行维度                                       | 列(均含环比列)                                                                                                                              |
| ---------------- | -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| 销售-整站        | 周期(最近 8 期,一行一期)                     | Orders / Total Sales / Returns / Net Sales / Sessions / Conversion rate                                                                     |
| 销售-SKU         | 业务SKU(副行 Collection·类别)                | Total sales / Returns / Net sales / Quantity ordered / Quantity returned / Net items sold / 日均销量                                        |
| 销售-Collections | Collection(由销售-SKU 事实行汇总)            | Total sales / Returns / Net sales / Quantity ordered / Quantity returned / Net items sold / 日均销量                                        |
| 渠道-整站        | Session source / medium(辅助列:渠道分组)     | Sessions / Session key event rate (Purchase) / Purchases / Total revenue                                                                    |
| 产品-渠道        | 业务SKU × source/medium(支持按 SKU 分组折叠) | Sessions / Items viewed / Items added to cart / Items purchased / Session key event rate / Item revenue                                     |
| 落地页-整站      | Landing page                                 | Sessions / Active users / New users / Average engagement time per session / Key events / Session key event rate / Total revenue / Purchases |

> 列口径调整:销售-整站不再展示 **Discounts**(折扣仍在底层 metrics,仅不展示);销售-SKU 去掉 **Sessions / Conversion rate**(原为待接入的空列),新增 **日均销量** = Net items sold ÷ 周期天数(周=7 天,月=当月天数),由服务端组表层计算。渠道-整站的 **Session key event rate (Purchase)** = 购买会话率(有购买的会话/全部会话),服务端用 GA4 `sessionKeyEventRate` + `eventName` 过滤 `session_start|purchase` 单独取数,不再是「所有关键事件」的会话率;历史周期需「补采」才会更新为新口径。前端表格列完全由服务端返回的 `columns` 驱动,增删列无需改前端。

**销售-SKU tab 下附「周×SKU」透视矩阵(3 张)**:在单期表下方渲染,行=最近 N 个有数据的周期(默认 8,时间降序、最近在最上,周期筛选同销售整站)、列=各业务SKU(动态列,按区间总销量降序),每个 SKU 单元格与「周总数」列均显示周度/月度环比。接口 `GET /tables/sku-trends`(`analyticsApi.getSkuTrends`),复用 `ReportTable`(动态 `columns`),列多时横向滚动。

| 矩阵表          | 值                                               | 说明                                                                                                       |
| --------------- | ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| SKU-销量-环比   | Net items sold(Shopify)                          | 首列后带「周总数」= Σ各SKU                                                                                 |
| SKU-销额-环比   | 北美 Net Sales / 非北美 Total Sales              | 由后端 `northAmericaShops` 配置判定本店(默认 filteroutlet / waterdropcanada 为北美);带「周总数」           |
| SKU-转化率-环比 | GA4 浏览→购买率 = Items purchased ÷ Items viewed | GA4 无法按 SKU 给出「Session key event rate(purchase)」(会话域),用商品域浏览→购买率替代;历史周期需补采才有 |

**销售-Collections tab 下附「周×Collection」透视矩阵(2 张)**:在单期 Collection 汇总表下方渲染,数据由服务端从 `SALES_SKU` 事实行按 `Collection` 实时聚合;行=最近 N 个有数据周期(默认 8,时间降序、最近在最上),列=Collection(动态列,按区间总销量降序),每个 Collection 单元格显示周度/月度环比 ↑↓。接口 `GET /tables/collection-trends`(`analyticsApi.getCollectionTrends`),复用 `ReportTable`;搜索框可过滤 Collection 行,矩阵里也按 Collection 列名过滤。

| 矩阵表               | 值                                  | 说明                                                                 |
| -------------------- | ----------------------------------- | -------------------------------------------------------------------- |
| Collection-销量-环比 | Net items sold(Shopify)             | 行按时间倒序,列为 Collection                                         |
| Collection-销额-环比 | 北美 Net Sales / 非北美 Total Sales | 由后端 `northAmericaShops` 配置判定本店;行按时间倒序,列为 Collection |

**渠道-整站 tab 下附「周×渠道」透视矩阵(3 张)**:在单期渠道整站表下方渲染,数据由服务端从 `CHANNEL_SITE` 事实行按 `Session source / medium` 实时展开;行=最近 N 个有数据周期(默认 8,时间降序、最近在最上),列=source/medium(动态列,按区间总 sessions 降序),每个渠道单元格显示周度/月度环比 ↑↓。接口 `GET /tables/channel-trends`(`analyticsApi.getChannelTrends`),复用 `ReportTable`;搜索框可过滤渠道行,矩阵里也按渠道列名过滤。

| 矩阵表             | 值                                     | 说明                              |
| ------------------ | -------------------------------------- | --------------------------------- |
| 渠道转化率-环比    | Session key event rate (Purchase)(GA4) | 行按时间倒序,列为 source / medium |
| 渠道流量绝对值变化 | Sessions(GA4)                          | 行按时间倒序,列为 source / medium |
| 渠道销额绝对值变化 | Total revenue(GA4)                     | 行按时间倒序,列为 source / medium |

**落地页-整站 tab**:数据来自 GA4 Landing page 报表,服务端按 `landingPage` 维度落 `LANDING_PAGE_SITE` 事实表;指标为 Sessions / Active users / New users / Average engagement time per session / Key events / Session key event rate / Total revenue / Purchases,并与其它维度表一致展示上一周期环比 ↑↓。表格默认按 Sessions 降序服务端分页加载 30 行/页,搜索框通过接口 `search` 参数先筛选 landing page 路径再分页;使用 Shopify `s-table` 官方分页控件,下方显示页码/总数提示;Landing page 列固定 450px。

**表格交互(共用 `ReportTable` 组件)**:

- 点列头排序(默认按主指标降序);维度表首列冻结;合计行置顶加粗。
- 环比列:`↑12.3%` 绿 / `↓8.1%` 红 / `—`(无对比期);tooltip 显示上期值。
- 销售-整站为多期趋势表:最新期在最上,可加「连续上涨/下降」徽标(后端 trend 字段)。
- 搜索框(销售-SKU / 销售-Collections / 渠道-整站 / 产品-渠道 / 落地页-整站):按业务SKU / Collection / source-medium / landing page 过滤;SKU/Collection/渠道矩阵同步支持按动态列名过滤。
- 「未映射」聚合行(如有):置底、黄色提示,点击跳映射管理。
- 右上角「导出 Excel」→ `GET /analytics/tables/:type/export`(带当前粒度/周期参数,`http.downloadFile`)。

**空状态**:该周期无事实数据 → 引导「补采本期数据」;从未配置 GA4 → 分步引导卡(① 设置页配 GA4 → ② 口径管理同步目录/排 Collection → ③ 回来补采)。

### 3.2 报告存档 `/app/analytics/reports` 与详情 `/$reportId`

**列表页**:

- 表格列:周期(W23 / 2026-05)、类型(周报/月报)、状态(`ReportStatusBadge`:PENDING/RUNNING/SUCCEEDED/FAILED)、生成时间、AI(有/无)、操作(查看 / 导出)。
- 「生成报告」按钮:选粒度+周期(默认上一期)→ `POST /analytics/reports`;列表存在 RUNNING 项时 5s 轮询(沿用竞品周报页模式)。

**详情页**:

- 元信息条:周期、数据截止日、生成时间、状态。
- **AI 结论卡**(`AiInsightCard`,`SimpleMarkdown` 渲染):趋势判断 / 问题定位 / 优化建议 / 渠道洞察 / SKU 洞察,纯文本分节展示。
- 六张表格快照(同 `ReportTable`,数据来自 `result.tables`,只读;旧报告未包含新表时显示空提示)。
- 未匹配清单(当期,一键跳映射管理)。
- 右上角「导出整份报告」→ `GET /analytics/reports/:id/export`(多 sheet xlsx)。

### 3.3 口径管理 `/app/analytics/maps`(v2.1,商品维度改为零人工)

页头放「同步目录」按钮(`POST /products/sync` + `/collections/sync`:从 Shopify 拉最新 product/metafield/collection)。`<s-tabs>` 三类,顺序:**商品口径 / 渠道分组 / Collection 管理**(Collection 为低频维护项,置于末位)。

#### Tab 1 · 商品口径(产品字典,只读为主 + 异常覆盖)

- 数据来自 `AnalyticsProduct`(`GET /analytics/products`):列 = 商品标题、productId、业务SKU(=metafield Model)、命中 Collection、GA4 对齐状态、状态、操作。
- **业务SKU 只读**:来自 Shopify product 的 metafield `Model`。缺失行标红「缺 Model」,引导**去 Shopify 给该 product 补 metafield**(数据源头修);确需临时适配,行尾「覆盖」弹窗填 `modelOverride`(`PATCH /products/:productId`)。
- **GA4 对齐**:itemId 自动解析 productId,状态显示「已对齐 / 未对齐」;未对齐行「修正 GA4」弹窗指定 productId 或直接业务SKU(`PATCH /ga4-items/:itemId`)。
- 筛选:全部 / 仅缺 Model / 仅 GA4 未对齐;搜索 product 标题/SKU。
- 顶部「同步目录」按钮(`POST /products/sync`)。

#### Tab 2 · 渠道分组(v2.1:分组本身可管理)

| 只读键列        | 可编辑列                                           |
| --------------- | -------------------------------------------------- |
| source / medium | 渠道分组(行内 `<s-select>`,选项来自分组字典)、状态 |

- **分组字典管理**:工具栏「管理分组」打开 `<s-modal>`,可新增 / 改名 / 删除分组(`GET/POST/PATCH/DELETE /analytics/channel-groups`)。映射行用 `channelGroupId` **引用**分组,故改名后引用自动跟随;删除分组时引用它的渠道自动置「未分组」。
- 未分组项下拉首选项展示「建议:XXX」(内置规则:`*/cpc→Paid Search`、`*/organic→Organic`、`chatgpt.com/perplexity.ai→AI Referral`…);筛选(全部/仅未分组/仅停用)、停用/启用。
- 内置默认分组(Paid Search/Organic/Paid Social/Direct/Referral/Email/AI Referral)首次初始化,之后完全由运营维护。

#### Tab 3 · Collection 管理(排序,低频)

- 列出站点全部 Collection(`GET /analytics/collections`),每行:拖拽手柄、序号、Collection 名、商品数、「置顶」。
- 排序语义:product 命中多个 collection 时,**取此处排序最靠前的命中项**作为展示 Collection。
- **交互**:整行**拖拽排序**(HTML5 draggable,Polaris 无拖拽组件,属自绘例外;支持键盘 ↑↓ 移动以满足 a11y)+ 行尾**置顶**;无停用操作。保存调 `PUT /analytics/collections/order`。
- 顶部「从 Shopify 同步 Collection」(`POST /collections/sync`);新 collection 追加到末尾,保留既有排序。

> v2.1 移除了原「Shopify SKU 映射」「GA4 商品映射」两个人工 Tab——商品维度现由 metafield + itemId 解析自动完成,口径管理只承担商品异常覆盖、渠道分组、Collection 排序。

### 3.4 设置 `/app/analytics/settings`(保留 v1 设计)

三块卡片,不变:

1. **GA4 配置**:Property ID 输入 + 「如何获取?」帮助气泡 + 「测试连接」(`POST /config/test-ga4`,即时反馈)。
2. **币种**:默认店铺币种(`shop.currencyCode`,只读;可覆盖下拉本期可不做)。

## 5. 通用状态与反馈规范

- **加载**:表格骨架屏(表头 + 占位行),不要白屏。
- **采集/生成中**:`useLoading` + 轮询(collect 轮询 `/periods` 或 job 状态;报告轮询 `GET /reports/:id`),到终态停止并 toast。
- **空状态**:每页给引导式空态(下一步该做什么)。
- **错误**:复用 `RetryErrorBanner`。
- **成功/失败**:`shopify.toast.show(...)`。
- **危险操作**(删除映射、replace 导入):二次确认。
- **导出**:`http.downloadFile`,导出中按钮 loading,完成 toast。

### API 契约(与后端方案 §9 一致)
